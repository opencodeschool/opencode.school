Quando o OpenCode edita ficheiros no teu projeto, queres saber exatamente o que mudou. O [Git](https://git-scm.com) dá-te essa visibilidade. É um sistema de controlo de versões que regista cada alteração a cada ficheiro de um projeto — o que foi adicionado, o que foi removido, e quando. É a ferramenta que os programadores usam para gerir código-fonte, e é a base de praticamente todos os projetos open-source.

Mas não precisas de ser programador nem de partilhar o teu trabalho com ninguém para beneficiar do Git. Mesmo que o uses apenas localmente, o Git dá-te um histórico completo do teu projeto. Cada commit é um snapshot ao qual podes regressar. Se o OpenCode fizer uma alteração que não te agrada, podes revertê-la. Nunca precisas de te preocupar em perder ou sobrescrever trabalho acidentalmente.

O [GitHub](https://github.com) é uma plataforma para alojar repositórios Git na cloud. Facilita a criação de cópias de segurança dos teus projetos, a sua partilha pública e a colaboração com outros. Em conjunto, o Git e o GitHub formam o conjunto de ferramentas padrão para gerir código — e funcionam igualmente bem para qualquer projeto com ficheiros que queiras acompanhar.

## Começar com o GitHub Desktop

A forma mais fácil de pôr o Git a funcionar é instalar o [GitHub Desktop](https://desktop.github.com/). Trata de três coisas de uma vez: instala o Git na tua máquina, cria uma conta GitHub se não tiveres uma, e autentica o CLI do Git para que comandos como `git push` funcionem automaticamente.

Se já usas o Git a partir da linha de comandos, podes saltar o GitHub Desktop por completo — o exercício funciona de qualquer forma.

## O que vais fazer

- **Inicializar um repositório** numa pasta de projeto para que o Git comece a acompanhar as alterações
- **Fazer commits** — guardar snapshots do teu trabalho com uma breve descrição do que mudou
- **Rever diffs** — usar o GitHub Desktop ou `git diff` para ver exatamente o que o OpenCode alterou nos teus ficheiros
- **Enviar para o GitHub** — fazer backup do teu repositório na cloud, público ou privado
- **Criar um branch** — experimentar algo sem afetar o teu projeto principal

Quando o básico estiver a funcionar, podes ir mais longe: escrever um `.gitignore` para manter o teu repositório limpo, abrir um pull request, ou explorar o histórico de commits do teu projeto em github.com.
