Neste exercício, vais usar o OpenCode para fazer brainstorming e criar rascunhos de publicações nas redes sociais, e depois enviá-los para o [Typefully](https://typefully.com) onde podes rever e publicá-los. O Typefully suporta cinco plataformas: X (Twitter), LinkedIn, Threads, Bluesky e Mastodon — podes publicar em qualquer combinação a partir de um único rascunho.

O fluxo de trabalho é conversacional: discutes ideias com o teu agente, refinam a redação em conjunto, e o agente cria o rascunho no Typefully. Nada é publicado sem a tua revisão.

## Pré-requisitos

Este exercício requer a **skill Typefully** para o OpenCode. Se completaste a [lição de Skills](/lessons/skills), já sabes como instalar skills. Executa:

```
npx skills add typefully/agent-skills
```

Também precisas de uma **conta Typefully** e chave de API:

1. Regista-te em [typefully.com](https://typefully.com) — é gratuito, e podes usar a tua conta Google, X ou LinkedIn existente
2. Liga pelo menos uma plataforma social (X, LinkedIn, Threads, Bluesky ou Mastodon)
3. Gera uma chave de API em [Definições > API](https://typefully.com/?settings=api)
4. Adiciona a chave ao teu perfil de shell (`~/.zshrc` ou `~/.bashrc`):

```
export TYPEFULLY_API_KEY=your_key_here
```

Reinicia o OpenCode depois de adicionares a chave para que a nova variável de ambiente seja reconhecida.

## O que vais fazer

**Brainstorming** — Diz ao teu agente sobre o que queres publicar. Pode ser algo que aprendeste, um projeto em que estás a trabalhar, uma opinião controversa, uma pergunta — qualquer coisa. O agente ajuda-te a dar forma à ideia e a escrever o texto.

**Rascunho** — O agente cria a tua publicação como rascunho no Typefully. Vais vê-la na aplicação web do Typefully, onde podes editar o texto, escolher em que plataformas publicar, e agendar para uma hora específica ou publicar imediatamente.

**Thread** (opcional) — Se tens uma ideia maior, experimenta criar uma thread — uma série de publicações ligadas que contam uma história ou explicam algo passo a passo. As threads funcionam bem no X e LinkedIn.

As publicações vão sempre primeiro para rascunhos, nunca diretamente para publicação. O Typefully é o passo de revisão onde decides o que vai para o ar e onde.

## Dicas de escrita

Boas publicações nas redes sociais tendem a partilhar alguns traços:

- Começa com um gancho — a primeira linha decide se alguém lê o resto
- Escreve como falas. Linguagem rígida e formal é rapidamente ignorada.
- Mantém o foco. Uma ideia por publicação. Se tens três ideias, são três publicações.
- Faz uma pergunta ou partilha uma opinião — publicações que convidam a uma resposta geram mais interação do que anúncios

O teu agente pode ajudar-te a reescrever e a apertar os teus rascunhos. Não te conformes com a primeira versão.

## Agendamento

O Typefully tem um agendador — podes dizer ao teu agente para agendar uma publicação para uma data e hora específicas em vez de a guardar apenas como rascunho. Isto é útil para publicar nas horas de maior afluência ou preparar o conteúdo de uma semana inteira de uma só vez.

## Ir mais longe

A API do Typefully consegue fazer mais do que criar rascunhos de texto. Algumas coisas para experimentar:

- Anexar imagens, vídeos, GIFs ou PDFs às tuas publicações
- Organizar rascunhos com etiquetas (útil se publicas sobre vários temas)
- Publicar o mesmo conteúdo em diferentes plataformas com texto específico para cada uma — por exemplo, uma versão mais curta para o X e uma mais longa para o LinkedIn
