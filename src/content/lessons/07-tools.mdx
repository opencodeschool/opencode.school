AI models don't just generate text — they can also call [tools](/glossary#tool). When you ask OpenCode to read a file, run a shell command, or search the web, the model is making a "tool call." It decides which tool to use, what arguments to pass, and then acts on the result. This is sometimes called "function calling," and it's what makes OpenCode an agent rather than a chatbot.

OpenCode comes with a set of built-in tools (reading files, editing code, running commands, and more). But you can also connect it to external tools through [MCP](/glossary#mcp) (Model Context Protocol) — an open standard for giving AI models access to additional capabilities. For example, you can add an MCP server that lets OpenCode search your GitHub issues, query a database, or look up documentation.
