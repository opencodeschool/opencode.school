agentInstructions: |
  This lesson is a short, optional interview. The student's answers are used to personalize the rest of the course. Every question can be skipped — make this clear up front.

  Before asking questions, auto-detect the student's operating system by inspecting the environment (e.g. check uname or the OS environment variable). Report it to the API immediately via PUT /api/profile/{studentId} with { "os": "macos" } (or "linux", "windows", etc.). Do not ask the student what OS they use.

  Present ALL of the following questions at once in a single message using multiple question tool calls in parallel. Include a "Skip" option in every question. This keeps things fast — the student answers everything in one pass instead of waiting between questions.

  After the student responds, check whether they answered "Builder" or "Sage" for the coding experience question. If so, ask one follow-up question about programming languages (see Question 7 below). If not, skip it.

  Question 1 — How much programming experience do you have?
  Single select. Options: Rookie (I've never written code before), Dabbler (I've tinkered a bit), Builder (I build things regularly), Sage (I live in the code).
  API value for codingExperience: "rookie", "dabbler", "builder", or "sage".

  Question 2 — Which AI tools have you used?
  Multi-select (select all that apply). Include "None of these" as an option. Use these labels and descriptions:
  - ChatGPT (OpenAI's chat assistant)
  - Gemini (Google's chat assistant)
  - Claude (Anthropic's chat assistant)
  - Claude Code (Terminal coding agent)
  - OpenAI Codex (Terminal coding agent)
  - GitHub Copilot (Editor autocomplete)
  API value for aiTools: array of "chatgpt", "gemini", "claude", "claude-code", "openai-codex", "copilot". Send an empty array if they select "None of these" or skip.

  Question 3 — What code editor do you use?
  Single select. Options: VS Code, Cursor, Windsurf, Zed, Vim, Emacs, Other, I don't have one yet.
  API value for editor: "vscode", "cursor", "windsurf", "zed", "vim", "emacs", "other", or "none".

  Question 4 — How comfortable are you with the terminal?
  Single select. Options: Not at all, I can get around, Very comfortable.
  API value for terminalComfort: "none", "some", or "very".

  Question 5 — How do you prefer to learn?
  Single select. Options: Explain concepts first, Jump into hands-on tasks, Show me examples.
  API value for learningStyle: "concepts-first", "hands-on", or "examples".

  Question 6 — How much detail do you want in explanations?
  Single select. Options: Keep it brief, Some context is nice, Give me all the details.
  API value for depthPreference: "brief", "some-context", or "all-details".

  Question 7 — What programming languages do you use? (CONDITIONAL — only ask as a follow-up)
  Only ask this if the student answered "Builder" or "Sage" for question 1. Skip it entirely for Rookie, Dabbler, or if question 1 was skipped.
  Multi-select (select all that apply). Options: JavaScript/TypeScript, Python, Go, Rust.
  API value for languages: array of "javascript", "typescript", "python", "go", "rust". For "JavaScript/TypeScript" send both "javascript" and "typescript".

  After all answers are collected, submit them in a single PUT /api/profile/{studentId} request. Include only the fields the student answered — omit skipped fields.

  Then briefly acknowledge their answers in a friendly way, referencing their preferences (e.g. "Got it — you're a Builder who prefers hands-on learning with concise explanations. I'll keep that in mind for the rest of the course."). If they skipped everything, that's fine — just say you'll teach at a general level.

  Note: This lesson happens before the student has configured permissions, so they may need to manually approve a few permission prompts from OpenCode (like allowing network requests). If the student asks about this, reassure them it's normal and will be addressed in the Configuration lesson.

  Mark the lesson complete after submitting the profile (or after acknowledging that all questions were skipped).

Nesta lição, o OpenCode vai fazer-lhe algumas perguntas opcionais sobre o seu percurso e preferências. As suas respostas ajudam a adaptar o resto do curso, ajustando o nível de detalhe, escolhendo exemplos relevantes e adequando-se ao seu nível de conforto com o terminal e o código.

Todas as perguntas são opcionais. Pode saltar as que preferir não responder.

Como estamos no início do curso, o OpenCode ainda não tem permissões alargadas. Poderá ter de clicar em "Allow" em alguns pedidos durante esta lição — isso é normal. As permissões serão configuradas numa lição futura.
