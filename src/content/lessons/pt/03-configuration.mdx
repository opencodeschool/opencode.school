## Criar o ficheiro de configuração global

Quando o OpenCode inicia, procura um ficheiro de configuração global nesta localização:

<div id="config-location-macos" class="hidden">

```
~/.config/opencode/opencode.json
```

O `~` é o seu diretório pessoal (`/Users/seunome`).

</div>

<div id="config-location-linux" class="hidden">

```
~/.config/opencode/opencode.json
```

O `~` é o seu diretório pessoal (`/home/seunome`).

</div>

<div id="config-location-windows" class="hidden">

```
~/.config/opencode/opencode.json
```

Isto fica dentro do seu diretório pessoal do WSL. No Windows nativo: `C:\Users\<SeuNome>\.config\opencode\opencode.json`.

</div>

<div id="config-location-other" class="hidden">

- **macOS / Linux**: `~/.config/opencode/opencode.json`
- **Windows (WSL)**: `~/.config/opencode/opencode.json` (dentro do seu diretório pessoal do WSL)
- **Windows (nativo)**: `C:\Users\<SeuNome>\.config\opencode\opencode.json`

O OpenCode irá criar o diretório e o ficheiro por si. **Se vir algum pedido de confirmação, escolha "Allow Once."** Poderá ver alguns pedidos durante este passo, por isso aprove cada um.

## Definir alguma configuração básica

Eis como o seu ficheiro de configuração ficará:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "default_agent": "plan",
  "permission": {
    "*": "allow",
    "external_directory": "ask",
    "bash": {
      "*": "allow",
      "rm *": "ask",
      "rmdir *": "ask"
    }
  },
  "instructions": [
    "https://<this-site>/api/instructions/<your-student-id>"
  ]
}
```

Vamos substituir o seu ID de estudante real quando criarmos o ficheiro. Algumas notas:

- `"default_agent": "plan"` significa que cada nova sessão começa em [modo Plan](/glossary#agent), um agente apenas de leitura que pode ler os seus ficheiros e discutir o seu projeto mas não fará alterações. Quando estiver pronto para agir, muda para Build. Vamos abordar isto em detalhe na lição [Agents](/agents).
- `"external_directory": "ask"` significa que o OpenCode irá pausar e perguntar antes de ler ou editar ficheiros fora do diretório do projeto atual. Este é o comportamento predefinido, mas defini-lo explicitamente torna a configuração auto-documentada.
- O bloco `"bash"` dentro de `"permission"` adiciona salvaguardas para comandos do terminal. A maioria dos comandos executa automaticamente, mas `rm` (apagar ficheiros) e `rmdir` (remover diretórios) irão pausar e pedir a sua aprovação primeiro. Vamos adicionar mais regras de permissão na próxima lição.
- O URL de instruções permite ao OpenCode obter a sua inscrição na escola no início de cada sessão, para que tenha o seu perfil de estudante disponível mesmo quando está a trabalhar noutro assunto.

## Reiniciar o OpenCode

As alterações de configuração entram em vigor na próxima vez que o OpenCode iniciar. Quando o ficheiro estiver criado, **feche o OpenCode Desktop** e reabra-o, depois volte a esta sessão de chat para que o OpenCode possa verificar a sua configuração.
