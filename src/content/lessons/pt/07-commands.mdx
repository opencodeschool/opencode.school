Se se encontra a escrever o mesmo tipo de prompt vezes sem conta — "resume este projeto", "revê as minhas alterações", "explica este ficheiro" — os comandos personalizados permitem guardar esse prompt e executá-lo com um único comando slash.

Escreva `/` na caixa de texto e verá uma lista de comandos disponíveis. Comandos integrados como `/new`, `/undo` e `/help` estão sempre presentes. Os comandos personalizados que criar aparecem na mesma lista.

## Duas formas de definir um comando

### Ficheiros Markdown

Crie um ficheiro `.md` em `~/.config/opencode/commands/` (global, disponível em todos os projetos) ou `.opencode/commands/` (por projeto). O nome do ficheiro torna-se o nome do comando.

`~/.config/opencode/commands/summarize.md`:

```markdown
---
description: Resumir o projeto atual
---
Dá-me uma visão geral deste projeto. O que é? Quais são os ficheiros e pastas principais? Para que parece servir?
```

Agora pode executar `/summarize` em qualquer sessão do OpenCode.

### JSON no opencode.json

Também pode definir comandos diretamente no seu ficheiro de configuração:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "command": {
    "summarize": {
      "description": "Resumir o projeto atual",
      "template": "Dá-me uma visão geral deste projeto. O que é? Quais são os ficheiros e pastas principais? Para que parece servir?"
    }
  }
}
```

Ambas as abordagens produzem o mesmo resultado. Os ficheiros Markdown são mais fáceis de ler e editar; o JSON é conveniente se quiser tudo num único ficheiro de configuração.

## Experimente: criar um comando /summarize

Peça ao OpenCode para criar o ficheiro `~/.config/opencode/commands/summarize.md` com este conteúdo:

```markdown
---
description: Resumir o projeto atual
---
Dá-me uma visão geral deste projeto. O que é? Quais são os ficheiros e pastas principais? Para que parece servir?
```

**Depois reinicie o OpenCode.** Os comandos são carregados no arranque, por isso novos ficheiros de comandos não aparecerão até reabrir a aplicação.

Quando voltar, escreva `/summarize` na caixa de texto — deverá vê-lo aparecer na lista de autocompletar. Selecione-o e prima Return para o executar.

## Passar argumentos

Adicione um marcador `$ARGUMENTS` ao seu template e pode passar contexto extra ao invocar o comando:

```markdown
---
description: Explicar um ficheiro
---
Explica o que $ARGUMENTS faz em linguagem simples.
```

Depois execute como `/explain src/index.ts` e `$ARGUMENTS` é substituído por `src/index.ts`.

## Mais opções

Os comandos suportam algumas outras opções no frontmatter: `agent` (executar o comando com um agente específico) e `model` (utilizar um modelo específico para este comando). Vamos abordar agentes numa lição posterior. Para a referência completa, consulte a [documentação de comandos](https://opencode.ai/docs/commands/).
